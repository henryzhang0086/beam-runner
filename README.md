# 独木闯关 · Beam Runner

一款基于「独木 + 重力翻面」的跑酷闯关游戏。零构建、纯前端（HTML5 Canvas + 原生 JS），
可直接双击 `index.html` 运行，也可一键部署到 GitHub Pages。

> 角色固定在画面左侧 1/3，独木不断向前滚动；前方不断出现高度 1–3 格的地刺，
> 独木上下散布金币。**点跳跃**过低刺、**连点两下跳跃**（二段跳）过双格刺、
> **3 格刺跳不过去、必须按换位翻到独木另一面**。捡道具、过机关、闯关、对决。

---

## 一、运行方式

- **最简单**：双击 `index.html`（脚本采用经典 `<script>` 顺序加载，`file://` 直接可用）。
- **本地服务器**（可选，体验更稳）：
  ```bash
  cd 闯关游戏
  python -m http.server 8000      # 然后浏览器打开 http://localhost:8000
  ```
- **部署**：将整个文件夹推到 GitHub Pages 即可（无需打包/依赖）。

支持桌面（键盘）与移动端（虚拟按键，自动检测触屏显示）。

## 二、操作

| 动作 | 键盘 | 触屏 |
|------|------|------|
| 跳跃 / 二段跳 | `空格` `↑` `W` | 右下大按钮 |
| 换位（翻面） | `Shift` `↓` `S` | 左下 ⇅ |
| 使用道具 | `E` `F` `回车` | 左下 ★ |
| 暂停 | `P` `Esc` | — |

**核心规则**：1 格刺点一下跳；2 格刺连点两下（二段跳）；3 格刺跳不过去，换位翻面躲；
有 3–4 次容错机会；独木上下金币计分。

## 三、模式

1. **无限模式** —— 越跑越快，永无止境。
   - *普通赛 / 道具赛* 两种。
   - 每 30 个障碍出现一段**金币奖励段**（无障碍）。
   - 积分到 **200 / 400 / 600** 触发一次性「密集 + 加速」困难突击。
2. **闯关模式** —— 5 个手工关卡，地图与机关各不相同：
   - 机关：**空中悬刺**、**吐火鸟**（周期喷火）、**升降墙 + 开关**、**下方开关触发上方坠物**、
     **道具门**（须用炸弹炸开，出现前有教程）。
   - 道具：🛡护盾 / »加速 / ♥加命 / ✸炸弹。
   - 通关解锁下一关，按失误数评 1–3 星。
3. **人机对决** —— 同一张地图限时比分，高者胜；死亡一次扣分；右上小窗观察 AI 对手。
   - 三档对手强度，公平由「同 seed → 完全相同地图」保证。

## 四、架构

零依赖、模块化经典脚本，命名空间挂在 `window.CG`。

```
index.html                 入口与 DOM 覆盖层（菜单/暂停/结算/教程/倒计时）
css/style.css              覆盖层 UI 与触屏按键样式
js/
  config.js                所有可调参数 + 道具/关卡内容数据 + 推导几何常量
  utils.js                 数学 / 确定性 RNG / AABB / 缓动 / 颜色
  engine/
    storage.js             localStorage 持久化（最高分/关卡进度/设置）
    input.js               键盘+触屏 → 3 动作（跳/翻/道具）边沿事件
    audio.js               纯过程化 Web Audio 音效 + 极简背景乐
    particles.js           对象池粒子系统（每 World 一份）
    camera.js              屏幕震动
    game.js                顶层状态机 + 固定步长循环 + 对决分屏渲染
  entities/
    player.js              玩家：单/二段跳（高度硬上限保证 3 格不可跳）+ 翻面 + 道具
    obstacle.js            地刺/空中刺/吐火鸟/升降墙/开关/坠物/道具门
    coin.js  item.js       金币 / 道具掉落
  world/
    background.js          视差背景（星点/山脊/光晕）
    levelgen.js            无限生成器（可解性保证）+ 闯关编译器
    world.js              单 playfield 模拟单元（物理/碰撞/计分/渲染到视口）
  ai/opponent.js           对决 AI 控制器（前瞻反应，三档可战胜）
  ui/hud.js  screens.js    画布内 HUD / DOM 菜单控制
  main.js                  引导：自适应 HiDPI 画布、子系统初始化
```

### 关键设计决策

- **一套引擎，三种模式**：`World` 是唯一的模拟单元；无限/闯关喂不同的 `source`；
  对决用两个**同 seed** 的 `World`（地图逐刺一致 = 公平），其一由 AI 驱动，分屏小窗呈现。
- **物理全部以「格」为单位**，与分辨率解耦；碰撞在格空间做 AABB，独木天然隔开上下两面。
- **「3 格刺跳不过」由物理保证**：跳跃高度有硬上限 `MAX_JUMP_CELLS = 2.82 < 3`，
  无论时机如何都跳不过 3 格刺，只能翻面 —— 不靠运气、不可被 timing 突破。
- **无限生成可解性**：维护「路径侧」，路径侧只放可跳障碍、需换面处放 3 格刺并**清理落点侧**
  前后窗口；非路径侧故意撒障碍，杜绝「翻到空侧躺赢」。
- **生成确定性**：难度只由「已生成距离」与 seed 决定，不读任何单局动态状态 →
  对决双方地图必然一致。

## 五、调参与测试

- 所有手感参数集中在 `js/config.js`（重力、跳跃初速、硬上限、间距、速度、分数、道具时长…）。
- 仓库内含一套 Node 无头测试（`harness.js`，见交付说明）验证：
  跳跃高度不变量、生成确定性、AI 长时间运行无异常、对决双方状态一致、闯关可达终点。
- 已用无头 Chrome 截图回归（菜单 / 无限 / 闯关三屏渲染正确）。

## 六、浏览器兼容

现代 Chromium / Firefox / Safari（含移动端）。音效首个用户手势后解锁（浏览器自动播放策略）。
